# Виджет лимитов Google AI Pro для macOS (SwiftBar)

Этот проект позволяет отображать текущие лимиты использования моделей (Gemini и Claude/GPT) для нескольких аккаунтов Google прямо в статус-баре (Menu Bar) macOS.

## Новые возможности (v1.5)
* **Переключение аккаунтов в 1 клик**: В меню виджета напротив неактивных аккаунтов отображается аккуратная кнопка-бадж `Войти`. Нажатие на неё безопасно закроет Antigravity IDE, переключит профиль авторизации в базе данных, автоматически запустит IDE обратно и обновит статус виджета.
* **Подсветка активного профиля**: Активный в данный момент в IDE аккаунт автоматически подсвечивается приятным зелёным цветом и обозначается круглой галочкой (`checkmark.circle.fill`).
* **Автоматическая генерация сессий**: Скрипт использует OAuth токены для локальной генерации корректных бинарных слепков сессии (`db_value` в protobuf формате). Больше не нужно вручную входить в каждый аккаунт внутри IDE для импорта сессии.

## Структура проекта
* `auth.py` — Скрипт авторизации. Используется для добавления новых аккаунтов и автоматической генерации их сессий.
* `fetcher.py` — Скрипт-сборщик квот. Получает актуальные данные от Google API и сохраняет их в `quotas.json`, а также поддерживает сессии в актуальном состоянии.
* `swiftbar_plugin.py` — Плагин для отображения данных в статус-баре через утилиту SwiftBar.
* `switch_account.py` — Служебный скрипт для безопасного закрытия IDE, обновления базы данных `state.vscdb` по Bundle ID и повторного запуска IDE.
* `generate_sessions.py` — Служебный скрипт для принудительной генерации сессий для всех аккаунтов в `accounts.json`.
* `accounts.json` — Файл с сохраненными токенами авторизации и сгенерированными слепками сессий (создается автоматически).
* `quotas.json` — Файл кэша лимитов (создается автоматически).

## Инструкция по настройке

### Шаг 1. Авторизация аккаунтов
Для каждого аккаунта, лимиты которого вы хотите отслеживать, выполните команду в терминале:

```bash
python3 auth.py
```

1. Скрипт автоматически запустит веб-страницу авторизации Google в вашем браузере.
2. Войдите в нужный аккаунт Google и подтвердите доступ к приложению.
3. Вернитесь в терминал. Скрипт попросит ввести **алиас (отображаемое имя)** для этого аккаунта (например: *Рабочий*, *Личный*, *Аккаунт 2*).
4. Токены и слепок сессии будут безопасно сгенерированы и сохранены в локальный файл `accounts.json`.

Повторите процедуру для всех ваших аккаунтов.

### Шаг 2. Первоначальный сбор данных
Запустите сборщик квот вручную один раз, чтобы создать первый кэш лимитов:

```bash
python3 fetcher.py
```
Этот скрипт обновит токены, сделает запросы к Google API, выведет отладочную информацию в консоль и запишет данные в `quotas.json`.

### Шаг 3. Подключение к SwiftBar
1. Откройте приложение **SwiftBar**.
2. Если вы запускаете его впервые, оно попросит выбрать папку для плагинов. Рекомендуется создать отдельную папку (например, `~/SwiftBarPlugins`) или использовать стандартную.
3. Создайте символическую ссылку на `swiftbar_plugin.py` в вашей папке плагинов SwiftBar, изменив имя файла, чтобы задать частоту обновления (например, `5m` для обновления каждые 5 минут):

```bash
ln -s "/Users/daniilchugunnikov/Desktop/work/gemini_limits_widget/swiftbar_plugin.py" "~/SwiftBarPlugins/gemini_limits.5m.py"
```
*(Замените `~/SwiftBarPlugins` на путь к вашей реальной папке плагинов).*

Готово! В вашем статус-баре появится иконка 🤖 с текущим расходом лимитов. При клике на нее откроется детальное меню по каждому аккаунту с графическими шкалами.

## Полезные функции виджета
* **Переключение в 1 клик**: Напротив каждого неактивного аккаунта отображается кнопка `Войти`. Нажатие на неё заменяет сессию в IDE на лету.
* **Автоматическое фоновое обновление**: Виджет сам запускает `fetcher.py` в фоновом режиме каждые 5 минут. Статус-бар при этом не зависает.
* **Ручное обновление**: В самом низу меню виджета есть кнопка `🔄 Обновить квоты сейчас` — нажатие на нее мгновенно запустит сбор данных и обновит статус-бар.
* **Быстрое добавление**: Кнопка `👤 Добавить/Обновить аккаунт` откроет терминал и запустит `auth.py`.
